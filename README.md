# SkillRanker

AI-powered ranking of the best Claude Code Agent Skills.

**Live site:** https://skillranker-ai.github.io/skillranker/

## What it does

SkillRanker automatically discovers, evaluates, and ranks Claude Code skills from GitHub. It produces a static website with ranked skills organized by domain.

## Architecture

```
Discovery (GitHub API) -> Evaluation (rules) -> AI Enrichment (Claude) -> Export (JSON) -> Site (Astro)
```

### Pipeline layers

| Layer | What | How |
|-------|------|-----|
| **Discovery** | Finds skills from GitHub search, awesome-lists | `backend/discover.py` |
| **Evaluation** | Scores maintenance, docs, completeness, adoption, structure | `backend/evaluate.py` |
| **Enrichment** | AI generates summaries, categories, quality scores | `backend/enrich.py` |
| **Export** | Builds `skills.json` for the static site | `backend/export.py` |
| **Site** | Astro static site deployed to GitHub Pages | `site/` |

### Scoring (0-100)

- **Maintenance** (20%): commit recency, releases, contributors
- **Documentation** (15%): SKILL.md quality, README, code examples
- **Completeness** (15%): references, scripts, examples, templates
- **Adoption** (10%): stars, forks, watchers (log-scaled)
- **Structure** (10%): license, tests, CI/CD, topics
- **AI Quality** (10%): clarity, coherence (Claude-assessed)
- **AI Usefulness** (10%): practical value (Claude-assessed)
- **AI Novelty** (10%): uniqueness (Claude-assessed)

## Quick start

```bash
# Install
pip install -r requirements.txt

# Run full pipeline
python -m backend.pipeline

# Or step by step
python -m backend.discover --source awesome
python -m backend.evaluate
python -m backend.enrich --limit 20
python -m backend.export

# Build site
cd site && npm install && npm run build
```

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `GITHUB_TOKEN` | Recommended | GitHub PAT for higher API rate limits |
| `ANTHROPIC_API_KEY` | For enrichment | Claude API key for AI scoring |

## Deployment

The site deploys automatically via GitHub Actions:
- **Daily at 6 AM UTC**: full pipeline run
- **On push to main**: rebuild if backend or site changed
- **Manual**: trigger via workflow_dispatch
